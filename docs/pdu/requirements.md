# Infineon PDU 要件整理

作成日: 2026-06-18  
更新日: 2026-06-23
入力: ユーザー要件および既存要件整理

## 1. 目的

マキタ BL4040 (A-69939) 40Vmax / 4.0Ah リチウムイオンバッテリー 2 個を入力とし、ARES X の足回り系、アーム系、Mini PC、Wi-Fi アンテナモジュールへ電源を分配する Power Distribution Unit (PDU) を作る。18V バッテリーは本構成では使用しない。

アームへの出力は 24V のままとし、足回りへの出力は 24V から 36V 程度へ変更される可能性を前提に設計する。Mini PC と Wi-Fi アンテナモジュールへの給電は同じ BL4040 2 個系に統一しつつ、E-Stop 対象外の常時給電系として扱う。

この PDU は単なる DC/DC 電源ではなく、入力保護、ホットスワップ、足回り・アームの独立 ON/OFF、遠隔制御、非常停止、常時給電系の保護分離、電流・電圧・温度監視、将来的な CAN 統合までを含む電源管理基板として整理する。部品選定は [parts_selection.md](./parts_selection.md)、BOM は [../../bom/pdu_bom.csv](../../bom/pdu_bom.csv) に分ける。

## 2. 前提

- 本 PDU は、University Rover Challenge (URC) に出場するために ARES Project が開発する新機体 ARES X へ搭載することを前提とする。
- ARES X はローバー本体とロボットアームを含む火星探査ローバーであり、本 PDU は足回り系、アーム系、常時給電系への電源供給、遮断、監視を担う。
- 入力電源はマキタ BL4040 (A-69939) 40Vmax / 4.0Ah リチウムイオンバッテリー 2 個とする。
- BL4040 は満充電時 40Vmax、平均動作電圧 36.0V、容量 4.0Ah、エネルギー 144Wh、メーカー測定基準による最大出力 1.5kW クラスとして扱う。
- マキタ 18V 系バッテリーは使用しない。
- 2 個のバッテリーは個別に入力保護、電圧監視、電流監視を行い、保護後に PDU の入力バスへ供給する。
- 2 個のバッテリー入力は、動作中に片方のバッテリーを交換できるホットスワップ対応とする。
- バッテリー合成は直列接続を前提にせず、各入力の hot-swap / eFuse / ideal diode ORing または同等の負荷分担経路で実装する。
- 足回り出力とアーム 24V 出力は、少なくとも STM32 MCU から独立 ON/OFF できる。
- アーム出力は 24V nominal とする。
- 足回り出力は初期 24V nominal とする。将来 36V 程度へ変更される可能性があるため、主電源、保護、スイッチ、電流センサ、コネクタは 36V 級への変更余地を残す。
- Mini PC と Wi-Fi アンテナモジュールへの給電は常時給電系とし、E-Stop 時も停止しない。
- 足回り系、アーム系、常時給電系は、ノイズ分離、負荷変動の局所化、片系統故障時の影響最小化を目的として、入力分岐、保護、フィルタ、GND リターン、配線経路を分ける。
- XBee 経由で遠隔 ON/OFF 指令を受ける。
- E-Stop はソフトウェア経由だけでなく、ハードウェア経路でも足回り・アームの高電力系を停止できる。
- E-Stop のハードウェア停止経路は、Mini PC、Wi-Fi アンテナモジュール、STM32 MCU、XBee、監視系の常時給電を遮断しない。
- 電流、電圧、温度、故障状態を STM32 MCU が監視し、XBee または将来の CAN へ送出できる。
- MCU は当プロジェクトで採用実績の多い STM32 を前提とする。
- 電源保護、スイッチ、CAN、センサは Infineon を基本とする。XBee、コネクタ、ヒューズ、受動部品、MCU は Infineon 縛りの対象外とする。

## 3. スコープ

### 含めるもの

- BL4040 2 個の入力保護
- ホットスワップ、突入電流制限、逆流防止、バッテリー間の不要な横流防止
- 足回り用 24V から 36V 程度の主電源生成または分配
- アーム用 24V 電源生成
- Mini PC / Wi-Fi アンテナモジュール用常時給電枝への保護済み入力分岐
- 足回り出力とアーム 24V の独立 ON/OFF
- E-Stop による足回り・アーム高電力系遮断
- E-Stop 中も維持する常時給電系の保護、監視、ノイズ分離
- 電流・電圧・温度・故障状態の監視
- XBee UART 通信
- 将来の CAN 通信
- STM32 MCU、常時電源、ウォッチドッグ、リセット監視

### 含めないもの

- 足回りモータドライバ本体
- アームモータドライバ本体
- Mini PC 本体の選定
- Wi-Fi アンテナモジュール本体の RF 設計、アンテナ選定、無線設定
- バッテリーパック内部 BMS の設計変更
- XBee 無線モジュール本体の設計
- 最終 PCB レイアウト、熱シミュレーション、EMC 試験仕様

## 4. システム構成

```text
Makita BL4040 Battery A, 40Vmax / 36V nominal / 4.0Ah
  -> fuse / hot-swap / reverse protection / telemetry
Makita BL4040 Battery B, 40Vmax / 36V nominal / 4.0Ah
  -> fuse / hot-swap / reverse protection / telemetry
        -> protected input bus
        -> E-Stop controlled high-power bus
              -> Foot 24V DC/DC or protected output, future 36V-class option
                    -> high-side switch -> Foot drive output
              -> Arm 24V DC/DC
                    -> high-side switch -> Arm 24V output
        -> always-on protected auxiliary bus, not E-Stop controlled
              -> Mini PC 19V power board / rail
              -> Wi-Fi antenna module power board / rail

Always-on control rail
  -> STM32 MCU
  -> XBee
  -> CAN transceiver
  -> telemetry / watchdog / E-Stop monitoring
```

足回りとアームの高電力系は E-Stop 対象とし、E-Stop 押下時に確実に停止する。アーム系は 24V のまま維持する。足回り系は初期 24V とし、将来 36V 程度へ変更される可能性があるため、DC/DC、保護 IC、ハイサイドスイッチ、電流センサ、コネクタ、配線は 36V 級への変更余地を残す。

Mini PC と Wi-Fi アンテナモジュールは、同じ BL4040 2 個系から給電するが、足回り・アームの E-Stop、負荷急変、短絡、スイッチングノイズ、片系統故障の影響を受けにくい常時給電枝として設計する。入力側には専用ヒューズ、eFuse または hot-swap、フィルタ、TVS、UVLO/OVP、電流制限を設け、GND リターンと配線経路も高電力系から分ける。

## 5. 機能要件

### 5.1 バッテリー入力

- マキタ BL4040 (A-69939) 40Vmax / 4.0Ah リチウムイオンバッテリー 2 個を入力とすること。
- マキタ 18V 系バッテリーを入力として使わないこと。
- 入力保護、電圧監視、DC/DC、スイッチ、電流センサは、BL4040 の満充電時 40Vmax と過渡サージを考慮した耐圧余裕を持つこと。
- BL4040 の平均動作電圧は 36.0V、エネルギーは 144Wh として電力量と運用時間を見積もること。
- BL4040 1 個あたりの出力上限は、メーカー測定基準による最大出力 1.5kW クラスを超えないようにすること。
- 入力電流制限は、固定電流値だけでなくバッテリー電圧を考慮した電力制限を併用し、初期設計では 36V nominal 時に 1 個あたり約 42A 相当を上限目安とする。
- 2 個の BL4040 が正常に負荷分担できる状態では、入力合計は最大 3.0kW クラス以内を上限目安とする。ただし各バッテリー入力は個別に 1.5kW クラス以内へ制限する。
- バッテリー A/B を個別に保護、監視できること。
- 片側バッテリーの挿抜、低電圧、過電流、短絡、逆接、過温度が、もう片側のバッテリーや下流系統へ破壊的に波及しないこと。
- 動作中に片方のバッテリーを抜き差ししても、残る片側バッテリーで常時給電系と安全監視を継続できること。
- ホットスワップ中は、残る片側 BL4040 の 1.5kW クラス出力上限を超えないよう、必要に応じて足回り・アーム高電力系を出力制限または安全停止できること。
- バッテリー挿入時の突入電流を制限すること。
- 入力過電圧、入力低電圧、過電流、短絡、過温度を検出できること。
- バッテリー A/B の電圧と電流を個別に取得できること。
- バッテリー入力ごとに電子的な Enable/Disable を持つこと。
- 電子保護だけに依存せず、物理ヒューズまたは同等の保護をバッテリー近傍に置くことを検討する。
- hot-swap / eFuse / ideal diode ORing または同等の負荷分担方式により、バッテリー間の横流、逆流、片側 fault の波及を防ぐこと。
- 過電圧保護は、40Vmax を正常上限として扱い、回生、ケーブルインダクタンス、E-Stop 遮断過渡によるサージを別途吸収できること。
- 低電圧遮断と復帰閾値は、BL4040 の内部 BMS 遮断より先に PDU が高電力系を制限または停止できるよう、実機測定で確定する。
- 物理ヒューズ、hot-swap / eFuse の電流制限値、コネクタ定格、ハーネス定格は、BL4040 1 個あたり 1.5kW クラス以内の電力制限と協調させる。

### 5.2 主電源生成

- 入力バッテリー電圧の変動範囲から、足回りとアームに必要な電源を生成または分配できること。
- アーム出力は 24V nominal とし、初期目標は 24V ±5% とする。
- 足回り出力は初期 24V nominal とし、初期目標は 24V ±5% とする。
- 足回り出力は将来 36V 程度へ変更される可能性を持つため、36V 級に変更する場合は電圧、許容差、制御方式、電流、発熱、保護協調を再設計すること。
- 足回り用電源とアーム用 24V 電源は系統を分けること。
- 各主電源系統の出力電圧、出力電流、故障状態を STM32 MCU が監視できること。
- 各主電源系統の立ち上げはシーケンス制御できること。
- 片方の DC/DC または出力保護の異常時は該当する足回りまたはアーム負荷を安全側へ停止し、他方の主電源系統は安全条件を満たす場合に継続できること。
- 足回り出力は初期 24V / 40A 級として部品を検討する。10 秒ピーク、100 ms ピークは最終設計前に実測または下流機器仕様から確定する。
- アーム 24V は、ARES X の複数アクチュエータ軸をまとめて供給する前提で、初期推定値を連続 30A、10 秒ピーク 60A、100 ms ピーク 90A とする。
- アーム 24V の初期推定値は、単一アクチュエータの最大電流を単純合算した値ではなく、同時駆動パターン、電流制限、デューティ、把持・起動・停止時のピークを見込んだ PDU 側の暫定設計値とする。
- 足回り、アーム、常時給電系を同時に動作させる場合でも、各 BL4040 入力が 1.5kW クラスを超えないよう入力電力を監視し、必要に応じて足回り・アーム高電力系から優先的に制限する。
- 最終設計前に、足回り・アームそれぞれの連続電流、10 秒ピーク、100 ms ピークを実測または下流機器仕様から再確認する。

### 5.3 常時給電系

- Mini PC と Wi-Fi アンテナモジュールへの給電は、足回り・アームと同じ BL4040 2 個系から分岐すること。
- 常時給電系は E-Stop 対象外とし、E-Stop 押下中も Mini PC と Wi-Fi アンテナモジュールへ給電を継続すること。
- 常時給電系は、足回り・アーム高電力系の短絡、過電流、負荷急変、スイッチングノイズ、E-Stop 遮断過渡の影響を最小化する保護設計を持つこと。
- 常時給電系には、専用ヒューズ、eFuse または hot-swap、入力フィルタ、TVS、UVLO/OVP、電流制限、fault 表示または telemetry を設けること。
- 常時給電系の故障が足回り・アーム高電力系や入力バッテリーへ破壊的に波及しないこと。
- Mini PC 用出力は 19V/5A とする。
- Wi-Fi アンテナモジュールは Ubiquiti UISP airMAX Bullet AC `B-DB-AC` とし、Wi-Fi アンテナモジュール用出力は 24V/0.5A とする。
- `B-DB-AC` は 20V から 27V DC 入力、最大消費電力 7W、24V Passive PoE に対応するため、PDU 側の常時給電枝は下流基板で 24V/0.5A を安定生成できる入力余裕と保護協調を持つこと。
- PDU から Mini PC / Wi-Fi アンテナモジュール用常時給電基板へ渡す枝は、専用ヒューズ、eFuse、フィルタ、TVS、power good / fault 信号を持つ保護済み枝とする。
- Mini PC と Wi-Fi アンテナモジュールのコネクタ、保護方式、fault/telemetry 信号の詳細は、下流の電源基板要件で定義する。

### 5.4 足回り・アーム電源 ON/OFF

- 足回り出力とアーム 24V を独立に ON/OFF できること。
- 足回り出力は初期 24V / 40A 級として扱い、将来 36V 程度へ変更する場合は電流定格と熱設計を見直すこと。
- アーム出力は複数の M3508 クラス BLDC アクチュエータ、ベースホライゾン、把持を含む総電流で扱うこと。
- アーム出力は初期推定として連続 30A、10 秒ピーク 60A、100 ms ピーク 90A を扱えること。
- 各出力に過電流、短絡、過温度の保護を持たせること。
- 各出力の電流監視を行えること。
- 電源投入順序は STM32 MCU の状態遷移に従うこと。
- 故障した DC/DC または出力だけを停止し、入力電源、E-Stop、MCU、通信、共通保護に異常がない場合は他系統を継続できる設計にする。

### 5.5 E-Stop

- E-Stop 押下時は足回り出力とアーム 24V 供給を停止すること。
- E-Stop は STM32 MCU 経由のソフトウェア停止に加え、保護 IC の Enable、DC/DC Enable、またはハイサイドスイッチ Enable を直接落とすハードウェア経路を持つこと。
- E-Stop 中も STM32 MCU、XBee、CAN transceiver、Mini PC、Wi-Fi アンテナモジュールの常時電源は残し、停止理由を送信または記録できるようにする。
- E-Stop の遮断過渡で常時給電系に過電圧、過大な電圧降下、リセット、通信断が起きにくいよう、入力フィルタ、保持容量、TVS、GND リターン、遮断順序を設計する。
- E-Stop 解除後は自動復帰する。ただし、入力電圧、足回り出力、アーム 24V 出力、電流、温度、保護 IC の fault 状態に異常がないことを確認する復帰シーケンスを通す。

### 5.6 遠隔 ON/OFF

- XBee UART 経由で以下を受け付けること。
  - 全体 ON/OFF
  - 足回り出力 ON/OFF
  - アーム 24V ON/OFF
  - 状態要求
  - 故障解除要求
- XBee の heartbeat timeout 初期値は 1 秒程度とし、通信途絶時は足回り・アームの高電力系を OFF にする。
- 通信途絶時も Mini PC と Wi-Fi アンテナモジュールの常時給電は維持する。
- 遠隔 ON 指令は E-Stop、過電流、過温度、低電圧などの安全条件を満たす場合にのみ有効とする。

### 5.7 監視・テレメトリ

STM32 MCU は少なくとも以下を監視する。

- Battery A 電圧
- Battery B 電圧
- Battery A 電流
- Battery B 電流
- 足回り出力電圧
- アーム 24V 電圧
- 足回り出力電流
- アーム 24V 電流
- 常時給電枝入力電圧
- 常時給電枝入力電流または fault 状態
- Mini PC 19V/5A 電源の power good または fault 状態
- `B-DB-AC` 用 24V/0.5A 電源の power good または fault 状態
- 保護 IC の fault/warn 状態
- 足回り電源系の fault/warn 状態
- アーム DC/DC の fault/warn 状態
- ハイサイドスイッチの fault/diagnosis 状態
- 基板温度、FET 近傍温度、または IC 内部温度

保護用途とログ用途は分ける。保護は高速な過電流検出を優先し、ログは数 % 程度の精度を初期目標とする。ログは、時刻、状態、電圧、電流、温度、故障理由を含む。

### 5.8 STM32 MCU 制御

STM32 MCU は以下の状態を持つ。

- `OFF`
- `PRECHECK`
- `PRECHARGE`
- `READY`
- `FOOT_ON`
- `ARM_ON`
- `RUN`
- `FAULT`
- `E_STOP`

各状態では、Enable 信号、故障監視、通信応答、復帰条件を明確に定義する。`E_STOP` 状態でも常時給電系の監視と通信は維持する。

### 5.9 CAN 統合

- 将来的に Power Board、Rover MCU、Arm MCU を CAN で接続できること。
- CAN 2.0 を初期対応、CAN FD は将来拡張として扱う。
- 初期ビットレートは 1Mbps とする。
- 終端抵抗、コネクタ、GND、シールド、ノード ID 割当を別途仕様化する。
- XBee と CAN のどちらを主通信路とするかはフェーズごとに分けてよい。

## 6. 非機能要件

- 電源投入時の初期状態は安全側、つまり足回り出力とアーム 24V 出力 OFF とする。
- 常時給電系は、入力保護条件を満たす場合に自動起動し、E-Stop 状態でも維持する。
- STM32 MCU 暴走時はウォッチドッグで安全側へ移行する。
- 主要な高電流パスは配線抵抗、銅箔厚、コネクタ定格、温度上昇を見積もる。
- 高電流スイッチング部は、熱設計と放熱経路を前提に部品パッケージを選ぶ。
- 足回り電源系、アーム 24V 系、常時給電系は、入力側の共通インピーダンス、GND リターン、放熱経路、スイッチングノードの配置を分け、片系統の負荷変動やノイズが他系統へ回り込みにくいレイアウトにする。
- 特に常時給電系は、足回り・アーム系の故障時にも Mini PC と Wi-Fi アンテナモジュールが不要にリセットしないよう、保護協調、フィルタ、保持容量、配線経路を設計する。
- 電流センス配線、PMBus/I2C、UART、CAN はノイズ源から距離を取り、GND リターンを整理する。
- E-Stop、ヒューズ、メインスイッチなど、人間が直接扱う安全機能はデバッグ用ジャンパに依存しない。
- 室温試作だけでなく、40 degC 以上の環境と密閉時の温度上昇を見込む。

## 7. 検証項目

- マキタ BL4040 A/B を接続した状態で、入力保護と監視が正常に起動する。
- BL4040 の 40Vmax 入力で、入力保護、DC/DC、スイッチ、電流センサ、コネクタに耐圧余裕がある。
- 各 BL4040 入力が 1.5kW クラスを超えないよう電力制限または出力制限が動作する。
- マキタ 18V 系バッテリーを前提にした入力範囲、ヒューズ定格、低電圧閾値が残っていないことを確認する。
- バッテリー A/B の片側挿抜、片側低電圧、片側 fault 時に破壊的な逆流や横流が発生しない。
- 片側 BL4040 のみで動作する場合、常時給電系と安全監視を維持しつつ、足回り・アーム高電力系が単一バッテリーの出力上限内へ制限または安全停止する。
- バッテリー挿抜時に過大な突入電流が発生しない。
- 足回り出力とアーム 24V を個別に ON/OFF できる。
- アーム 24V が 24V ±5% の初期目標範囲に入る。
- 足回り出力が、初期 24V ±5% と 40A 級負荷条件で許容範囲に入る。
- アーム 24V が、連続 30A、10 秒ピーク 60A、100 ms ピーク 90A の初期推定負荷で許容範囲に入る。
- 足回り負荷変動時に、アーム 24V と常時給電系が許容電圧範囲を外れない。
- アーム 24V の起動・停止・把持ピーク時に、足回り出力と常時給電系が許容電圧範囲を外れない。
- 足回り電源系統の故障時に、安全条件を満たす場合はアーム 24V と常時給電系を継続できる。
- アーム 24V 系統の故障時に、安全条件を満たす場合は足回り出力と常時給電系を継続できる。
- 常時給電系の短絡または過電流時に、足回り・アーム高電力系と入力バッテリーへ破壊的に波及しない。
- E-Stop 押下で足回り・アームの高電力系が停止する。
- E-Stop 押下中も Mini PC、`B-DB-AC`、STM32 MCU、XBee が給電を維持する。
- E-Stop 解除後、電圧・電流・温度・fault 状態に異常がない場合のみ自動復帰する。
- STM32 MCU をリセットしても足回り・アーム高電力系が勝手に ON にならない。
- XBee から状態取得と ON/OFF 操作ができる。
- XBee heartbeat が約 1 秒途絶した場合、足回り・アーム高電力系が OFF になる。
- 過電流、短絡、過温度、低電圧を検出して FAULT 状態へ移行する。
- CAN 1Mbps でノード状態、電圧、電流、故障情報を送信できる。

## 8. 実装フェーズ

1. 要件確定
   - BL4040 2 個の hot-swap / ORing 構成、実測低電圧閾値、足回り出力電圧、足回り・アーム電流、常時給電枝の仕様を確定する。
2. BL4040 入力試作
   - ホットスワップ、突入電流、逆流防止、横流防止、電圧・電流監視を確認する。
3. 常時給電枝試作
   - Mini PC / Wi-Fi アンテナモジュール用電源基板への分岐、保護協調、E-Stop 非連動、ノイズ耐性を確認する。
4. 足回り・アーム電源試作
   - 足回り用電源とアーム 24V 電源それぞれで、入力変動、負荷変動、発熱、効率を確認する。
   - 片系統の負荷変動と故障が他系統および常時給電系へ与える影響を確認する。
5. 足回り・アーム出力
   - 独立 ON/OFF、過電流保護、故障診断を確認する。
6. E-Stop とフェイルセーフ
   - ハードウェア停止経路、常時給電系の維持、復帰手順、STM32 MCU 暴走時の挙動を確認する。
7. 通信統合
   - XBee での遠隔操作とテレメトリを実装し、CAN へ拡張する。
