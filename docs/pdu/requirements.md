# Infineon PDU 要件整理

作成日: 2026-06-18  
入力: ../../memo.md および加筆修正済み要件回答

## 1. 目的

マキタ 18V バッテリー 2 本を入力とし、ローバーの足回り系とアーム系へ 24V 電源を供給する Power Distribution Unit (PDU) を作る。単なる DC/DC 電源ではなく、ホットスワップ、独立 ON/OFF、遠隔制御、非常停止、電流・電圧・温度監視、将来的な CAN 統合までを含む電源管理基板として整理する。

この文書では、`memo.md` に含まれる構想に加え、本書へ追記された要件回答を本文へ反映して整理する。部品選定は [parts_selection.md](./parts_selection.md)、BOM は [../../bom/pdu_bom.csv](../../bom/pdu_bom.csv) に分ける。

## 2. 前提

- 本 PDU は、University Rover Challenge (URC) に出場するために ARES Project が開発する新機体 ARES X へ搭載することを前提とする。
- ARES X はローバー本体とロボットアームを含む火星探査ローバーであり、本 PDU は足回り系とアーム系への電源供給、遮断、監視を担う。
- 入力電源はマキタ 18V 系バッテリーを 2 本使用する。
- 2 本のバッテリーは、直列化ではなく独立入力として扱い、ホットスワップと冗長運用を目指す。
- 足回り 24V 系とアーム 24V 系は、少なくとも STM32 MCU から独立 ON/OFF できる。
- 足回り系とアーム系は、ノイズ分離、負荷変動の局所化、片系統故障時の継続運用を目的として、24V DC/DC を 2 系統に分ける。
- XBee 経由で遠隔 ON/OFF 指令を受ける。
- E-Stop はソフトウェア経由だけでなく、ハードウェア経路でも高電力系を停止できる。
- 電流、電圧、温度、故障状態を STM32 MCU が監視し、XBee または将来の CAN へ送出できる。
- MCU は当プロジェクトで採用実績の多い STM32 を前提とする。
- 電源保護、スイッチ、CAN、センサは Infineon を基本とする。XBee、コネクタ、ヒューズ、受動部品、MCU は Infineon 縛りの対象外とする。

## 3. スコープ

### 含めるもの

- バッテリー入力保護
- ホットスワップ、突入電流制限、逆流防止
- 足回り用 24V 電源生成
- アーム用 24V 電源生成
- 足回り 24V とアーム 24V の独立 ON/OFF
- E-Stop による高電力系遮断
- 電流・電圧・温度・故障状態の監視
- XBee UART 通信
- 将来の CAN 通信
- STM32 MCU、常時電源、ウォッチドッグ、リセット監視

### 含めないもの

- 足回りモータドライバ本体
- アームモータドライバ本体
- バッテリーパック内部 BMS の設計変更
- XBee 無線モジュール本体の設計
- 最終 PCB レイアウト、熱シミュレーション、EMC 試験仕様

## 4. システム構成

```text
Makita 18V Battery A
  -> hot-swap / protection / telemetry
Makita 18V Battery B
  -> hot-swap / protection / telemetry
        -> protected input bus
        -> Foot 24V DC/DC
              -> Foot 24V bus
              -> high-side switch -> Foot drive 24V
        -> Arm 24V DC/DC
              -> Arm 24V bus
              -> high-side switch -> Arm 24V

Always-on rail
  -> STM32 MCU
  -> XBee
  -> CAN transceiver
  -> telemetry / watchdog / E-Stop monitoring
```

24V 生成は、足回り用 24V とアーム用 24V の 2 系統に分ける。足回りの大電流変動やモータ系ノイズをアーム側へ伝えにくくし、アーム側の起動・把持・停止時の負荷変動も足回り側へ戻しにくくする。また、片方の DC/DC または出力系統が故障した場合でも、安全条件を満たす範囲で他方の 24V 系を継続運用できる構成を目指す。

## 5. 機能要件

### 5.1 バッテリー入力

- バッテリー A/B を独立入力として扱えること。
- 片側バッテリーの挿抜時に、もう片側の電源系へ破壊的な逆流を起こさないこと。
- バッテリー挿入時の突入電流を制限すること。
- 入力過電圧、入力低電圧、過電流、短絡、過温度を検出できること。
- バッテリー A/B の電圧と電流を個別に取得できること。
- バッテリー入力ごとに電子的な Enable/Disable を持つこと。
- 電子保護だけに依存せず、物理ヒューズまたは同等の保護をバッテリー近傍に置くことを検討する。

### 5.2 24V 生成

- 入力バッテリー電圧の変動範囲から、足回りとアームに必要な 24V を生成できること。
- 24V DC/DC は足回り用とアーム用の 2 系統に分けること。
- 各 24V 系統の出力電圧、出力電流、故障状態を STM32 MCU が監視できること。
- 各 24V 系統の立ち上げはシーケンス制御できること。
- 片方の DC/DC の異常時は該当する足回りまたはアーム負荷を安全側へ停止し、他方の 24V 系統は安全条件を満たす場合に継続できること。
- 24V 出力は各系統とも 24V ±5% とする。
- 足回り 24V は暫定 40A 級として部品を選定する。
- アーム 24V は、ARES X の複数アクチュエータ軸をまとめて供給する前提で、連続電流、10 秒ピーク、100 ms ピークを再見積もりする。単一アクチュエータの 10A から 20A 級だけを根拠に PDU 側を決めない。
- 最終設計前に、足回り・アームそれぞれの連続電流、10 秒ピーク、100 ms ピークを確定する。

### 5.3 足回り・アーム電源 ON/OFF

- 足回り 24V とアーム 24V を独立に ON/OFF できること。
- 足回り出力は暫定 40A 級として扱うこと。
- アーム出力は複数の M3508 クラス BLDC アクチュエータ、ベースホライゾン、把持を含む総電流で扱うこと。
- 各出力に過電流、短絡、過温度の保護を持たせること。
- 各出力の電流監視を行えること。
- 電源投入順序は STM32 MCU の状態遷移に従うこと。
- 故障した DC/DC または出力だけを停止し、入力電源、E-Stop、MCU、通信、共通保護に異常がない場合は他系統を継続できる設計にする。

### 5.4 E-Stop

- E-Stop 押下時は足回りとアームへの 24V 供給を停止すること。
- E-Stop は STM32 MCU 経由のソフトウェア停止に加え、保護 IC の Enable、DC/DC Enable、またはハイサイドスイッチ Enable を直接落とすハードウェア経路を持つこと。
- E-Stop 中も STM32 MCU と XBee の常時電源は残し、停止理由を送信できるようにする。
- E-Stop 解除後は自動復帰する。ただし、入力電圧、足回り/アーム各 24V 出力、電流、温度、保護 IC の fault 状態に異常がないことを確認する復帰シーケンスを通す。

### 5.5 遠隔 ON/OFF

- XBee UART 経由で以下を受け付けること。
  - 全体 ON/OFF
  - 足回り 24V ON/OFF
  - アーム 24V ON/OFF
  - 状態要求
  - 故障解除要求
- XBee の heartbeat timeout 初期値は 1 秒程度とし、通信途絶時は足回り・アーム 24V を OFF にする。
- 遠隔 ON 指令は E-Stop、過電流、過温度、低電圧などの安全条件を満たす場合にのみ有効とする。

### 5.6 監視・テレメトリ

STM32 MCU は少なくとも以下を監視する。

- Battery A 電圧
- Battery B 電圧
- Battery A 電流
- Battery B 電流
- 足回り 24V 電圧
- アーム 24V 電圧
- 足回り 24V 電流
- アーム 24V 電流
- 保護 IC の fault/warn 状態
- 足回り DC/DC の fault/warn 状態
- アーム DC/DC の fault/warn 状態
- ハイサイドスイッチの fault/diagnosis 状態
- 基板温度、FET 近傍温度、または IC 内部温度

保護用途とログ用途は分ける。保護は高速な過電流検出を優先し、ログは数 % 程度の精度を初期目標とする。ログは、時刻、状態、電圧、電流、温度、故障理由を含む。

### 5.7 STM32 MCU 制御

STM32 MCU は以下の状態を持つ。

- `OFF`
- `PRECHECK`
- `PRECHARGE`
- `READY`
- `FOOT_ON`
- `ARM_ON`
- `RUN`
- `FAULT`
- `E_STOP`

各状態では、Enable 信号、故障監視、通信応答、復帰条件を明確に定義する。

### 5.8 CAN 統合

- 将来的に Power Board、Rover MCU、Arm MCU を CAN で接続できること。
- CAN 2.0 を初期対応、CAN FD は将来拡張として扱う。
- 初期ビットレートは 1Mbps とする。
- 終端抵抗、コネクタ、GND、シールド、ノード ID 割当を別途仕様化する。
- XBee と CAN のどちらを主通信路とするかはフェーズごとに分けてよい。

## 6. 非機能要件

- 電源投入時の初期状態は安全側、つまり足回り/アーム 24V 出力 OFF とする。
- STM32 MCU 暴走時はウォッチドッグで安全側へ移行する。
- 主要な高電流パスは配線抵抗、銅箔厚、コネクタ定格、温度上昇を見積もる。
- 高電流スイッチング部は、熱設計と放熱経路を前提に部品パッケージを選ぶ。
- 足回り用 24V DC/DC とアーム用 24V DC/DC は、入力側の共通インピーダンス、GND リターン、放熱経路、スイッチングノードの配置を分け、片系統の負荷変動やノイズが他系統へ回り込みにくいレイアウトにする。
- 電流センス配線、PMBus/I2C、UART、CAN はノイズ源から距離を取り、GND リターンを整理する。
- E-Stop、ヒューズ、メインスイッチなど、人間が直接扱う安全機能はデバッグ用ジャンパに依存しない。
- 室温試作だけでなく、40 degC 以上の環境と密閉時の温度上昇を見込む。

## 7. 検証項目

- バッテリー A のみで足回り/アーム 24V 系が起動する。
- バッテリー B のみで足回り/アーム 24V 系が起動する。
- 片側バッテリー挿抜時に過大な突入電流や逆流が発生しない。
- 足回り 24V とアーム 24V を個別に ON/OFF できる。
- 足回り 24V の負荷変動時に、アーム 24V が許容電圧範囲を外れない。
- アーム 24V の起動・停止・把持ピーク時に、足回り 24V が許容電圧範囲を外れない。
- 足回り 24V 系統の故障時に、安全条件を満たす場合はアーム 24V 系統を継続できる。
- アーム 24V 系統の故障時に、安全条件を満たす場合は足回り 24V 系統を継続できる。
- E-Stop 押下で足回り・アームの高電力系が停止する。
- E-Stop 解除後、電圧・電流・温度・fault 状態に異常がない場合のみ自動復帰する。
- STM32 MCU をリセットしても高電力系が勝手に ON にならない。
- XBee から状態取得と ON/OFF 操作ができる。
- XBee heartbeat が約 1 秒途絶した場合、足回り・アーム 24V が OFF になる。
- 過電流、短絡、過温度、低電圧を検出して FAULT 状態へ移行する。
- CAN 1Mbps でノード状態、電圧、電流、故障情報を送信できる。

## 8. 実装フェーズ

1. 要件確定
   - 足回り・アーム 24V の連続電流とピーク電流を確定する。
2. 単一バッテリー入力試作
   - ホットスワップ、突入電流、電圧・電流監視を確認する。
3. 2 バッテリー入力試作
   - ホットスワップ、逆流防止、片側挿抜を確認する。
4. 2 系統 24V 生成試作
   - 足回り用 DC/DC とアーム用 DC/DC それぞれで、入力変動、負荷変動、発熱、効率を確認する。
   - 片系統の負荷変動と故障が他系統へ与える影響を確認する。
5. 足回り・アーム出力
   - 独立 ON/OFF、過電流保護、故障診断を確認する。
6. E-Stop とフェイルセーフ
   - ハードウェア停止経路、復帰手順、STM32 MCU 暴走時の挙動を確認する。
7. 通信統合
   - XBee での遠隔操作とテレメトリを実装し、CAN へ拡張する。
