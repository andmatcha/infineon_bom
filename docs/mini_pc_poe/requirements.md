# Mini PC / Wi-Fi アンテナ電源基板 要件整理

作成日: 2026-06-18  
更新日: 2026-06-23
入力: ユーザー要件および既存要件整理

## 1. 目的

Mini PC と Wi-Fi アンテナモジュールへ常時給電する電源基板を作る。入力は、ローバーの足回り系・アーム系と同じマキタ BL4040 (A-69939) 40Vmax / 4.0Ah リチウムイオンバッテリー 2 個系から分岐する。18V バッテリーや別系統バッテリーは使用しない。

本基板は E-Stop 対象外とし、E-Stop 押下中も Mini PC と Wi-Fi アンテナモジュールへの給電を維持する。ローバー足回り系とロボットアーム系の電源ノイズ、負荷急変、短絡、故障、E-Stop 遮断過渡の影響を最小限にするため、入力保護、フィルタ、GND リターン、配線分離、fault 隔離を重視する。

Mini PC 実機の電源入力は 19V/3.42A (64.98W) であり、基板側は起動突入と機種変更余裕を見込んで 19V/5A 定格として設計する。Wi-Fi アンテナモジュールは Ubiquiti UISP airMAX Bullet AC `B-DB-AC` とし、24V Passive PoE で給電する。`B-DB-AC` の入力範囲は 20V から 27V DC、最大消費電力は 7W であるため、基板側は 24V/0.5A 定格として余裕を持たせる。

この文書では、PDU 要件整理と同様に、現時点で決まっている要件を本文へ反映して整理する。部品選定は [parts_selection.md](./parts_selection.md)、BOM は [../../bom/mini_pc_poe_bom.csv](../../bom/mini_pc_poe_bom.csv) に分ける。

## 2. 前提

- 本基板は、Mini PC および Wi-Fi アンテナモジュールへ給電する常時給電基板である。
- 入力電源は、PDU の保護済み常時給電枝から受ける。
- PDU の元入力はマキタ BL4040 (A-69939) 40Vmax / 4.0Ah リチウムイオンバッテリー 2 個とする。
- マキタ 18V 系バッテリーや Mini PC 専用の別バッテリーは使用しない。
- ローバーの足回り系、アーム系とは入力バッテリーを共有するが、E-Stop、出力スイッチ、保護、フィルタ、GND リターン、配線経路は常時給電系として分離する。
- 出力は、基板定格 19V/5A の Mini PC 用電源と、24V/0.5A の Wi-Fi アンテナモジュール用電源の 2 系統とする。
- Mini PC 実機の電源入力は 19V/3.42A、センタープラス、外径 5.5mm / 内径 2.5mm の DC ジャックである。
- 両出力は常時給電とし、PDU の常時給電枝が保護条件を満たす場合に自動で出力する。
- 両出力は E-Stop 押下中も停止しない。
- Wi-Fi アンテナモジュールは Ubiquiti UISP airMAX Bullet AC `B-DB-AC` とする。
- Wi-Fi アンテナモジュール用電源は 24V/0.5A とする。`B-DB-AC` の電源仕様は 20V から 27V DC、最大 7W、24V DC / 0.3A 級 PoE アダプタであるため、0.5A 定格は定常消費に対して余裕を持つ。
- Wi-Fi アンテナモジュール用の接続方式は 24V Passive PoE とする。
- Passive PoE は IEEE 802.3af/at/bt 互換ではなく、検出・分類・ネゴシエーションを行わない固定 24V 給電とする。
- Passive PoE では、RJ45 の 1,2,3,6 ピンはデータ用として Data In と PoE Out の間を直結し、PoE Out 側の 4,5 ピンへ +24V、7,8 ピンへ GND を割り当てる。
- `B-DB-AC` 本体の Ethernet interface は Gigabit Ethernet だが、本基板の初期 Passive PoE 配線は 10BASE-T / 100BASE-TX の spare-pair 給電を前提とする。1000BASE-T で運用する場合は、全 4 ペアのデータ伝送と Passive PoE を両立できるギガビット対応 injector 構成へ別途変更する。
- 電源保護、出力スイッチ、パワー MOSFET は Infineon を基本候補とする。DC/DC コントローラ、RJ45、コネクタ、ヒューズ、TVS、受動部品は Infineon 縛りの対象外とする。

## 3. スコープ

### 含めるもの

- PDU の保護済み常時給電枝からの入力
- 入力ヒューズ、eFuse または hot-swap、逆接保護、突入電流制限、低電圧遮断、過電圧保護
- 足回り・アーム高電力系からのノイズ、短絡、故障、E-Stop 過渡を受けにくくする入力フィルタと保護協調
- 19V/5A Mini PC 用 DC/DC
- Wi-Fi アンテナモジュール用 DC/DC
- 24V/0.5A Wi-Fi アンテナモジュール出力
- 24V Passive PoE 出力
- Mini PC 出力および Wi-Fi アンテナモジュール出力の過電流、短絡、過温度保護
- Passive PoE の RJ45 Data In / PoE Out 配線、ESD 保護、シールド処理
- 電源状態確認用 LED、fault 信号、テストポイント
- 入出力コネクタ、ヒューズ、TVS、電源インジケータ

### 含めないもの

- Mini PC 本体の選定
- Wi-Fi アンテナモジュール本体の RF 設計、アンテナ選定、無線設定
- OS のシャットダウン制御
- IEEE 802.3af/at/bt 準拠の PoE PSE 機能
- 1000BASE-T 以上の Ethernet pass-through
- 足回り・アーム高電力系の E-Stop 主遮断
- 最終 PCB レイアウト、熱シミュレーション、EMC 試験仕様

## 4. システム構成

```text
PDU always-on protected auxiliary bus
  from Makita BL4040 Battery x2 system, 40Vmax / 36V nominal per pack
  not E-Stop controlled
  -> input fuse / eFuse or hot-swap / reverse protection / UVLO / OVP
  -> input EMI filter / TVS / bulk capacitor
        -> 19V DC/DC
              -> high-side protection switch
              -> Mini PC 19V / 5A output
        -> Wi-Fi antenna module DC/DC
              -> high-side protection switch / fuse
              -> Wi-Fi antenna module 24V / 0.5A output

Passive PoE output
  Ethernet Data In RJ45
    pin 1 -> PoE Out pin 1
    pin 2 -> PoE Out pin 2
    pin 3 -> PoE Out pin 3
    pin 6 -> PoE Out pin 6

  PoE Out RJ45
    pins 4,5 -> +24V
    pins 7,8 -> GND
```

本基板はローバーの足回り・アーム系と入力バッテリーを共有するが、E-Stop 対象外の常時給電枝に置く。PDU 側で足回り・アーム高電力系が停止、短絡、過電流、負荷急変しても、Mini PC と Wi-Fi アンテナモジュールの電源が不要に遮断またはリセットしないことを重視する。

## 5. 機能要件

### 5.1 入力

- PDU の保護済み常時給電枝を入力とすること。
- PDU の元入力はマキタ BL4040 (A-69939) 40Vmax / 4.0Ah リチウムイオンバッテリー 2 個とすること。
- マキタ 18V 系バッテリーや Mini PC 専用の別バッテリーを入力としないこと。
- 入力電圧範囲は、PDU 側の BL4040 合成方式、低電圧遮断、過電圧保護に合わせて確定すること。初期設計では BL4040 の 36V nominal / 40Vmax 系からの保護済み常時給電枝を前提にする。
- 旧 18V 系前提の 15V から 21V 入力範囲を設計前提として使わないこと。
- 入力に物理ヒューズを置くこと。
- eFuse、hot-swap、逆接保護、過電流、短絡、過温度、低電圧、過電圧を検出または遮断できること。
- 入力フィルタ、TVS、bulk capacitor、GND リターン設計により、足回り・アーム高電力系の負荷急変、E-Stop 遮断、スイッチングノイズの影響を最小化すること。
- 入力側 fault が PDU の常時給電枝や主バッテリー系へ破壊的に波及しないこと。
- 入力電圧、入力電流、fault 状態を LED、テストポイント、または将来 telemetry として確認できること。
- PDU から本基板へ渡す入力は、専用ヒューズ、eFuse、入力フィルタ、TVS、power good / fault 信号を持つ保護済み枝とする。

### 5.2 Mini PC 19V 出力

- 出力電圧は 19V nominal とする。
- 出力電流は基板定格として連続 5A とする。接続する Mini PC 実機の定格負荷は 3.42A とする。
- 出力電圧許容差は Mini PC の要求仕様を優先する。初期目標は 19V ±5% とする。
- DC/DC トポロジは確定した入力電圧範囲に合わせて選ぶこと。40V 系入力では降圧を基本候補とし、入力範囲が 19V をまたぐ場合のみ buck-boost を検討する。
- Mini PC 接続時の突入電流と出力容量充電を許容できること。
- 出力短絡、過電流、過温度、過電圧時に破壊しないこと。
- E-Stop 押下、足回り・アーム出力遮断、足回り負荷急変時にも 19V 出力が不要に落ち込まないこと。
- Mini PC 側コネクタは外径 5.5mm / 内径 2.5mm の DC ジャック、センタープラスとし、基板側からは同寸法のセンタープラス DC プラグ付きケーブルで給電すること。

### 5.3 Wi-Fi アンテナモジュール出力

- Wi-Fi アンテナモジュール `B-DB-AC` へ 24V nominal、連続 0.5A を供給すること。
- `B-DB-AC` は 20V から 27V DC 入力、最大消費電力 7W、24V Passive PoE に対応するため、24V/0.5A 出力を初期定格とする。
- Wi-Fi アンテナモジュールの接続方式は Passive PoE とする。
- `B-DB-AC` の起動突入電流は公開仕様だけでは確定しないため、初期試作で実測し、出力電流制限、ソフトスタート、出力容量、ヒューズ/PTC 定格へ反映すること。
- 24V Passive PoE では、初期設計でケーブル長 10m から 20m 程度を評価対象とする。
- 24V Passive PoE では、24V 出力は PoE Out RJ45 の 4,5 ピンへ +24V、7,8 ピンへ GND を供給すること。
- Passive PoE では、Data In RJ45 と PoE Out RJ45 の 1,2,3,6 ピンは、差動ペアとしてインピーダンスとペア関係を保って直結すること。
- Passive PoE では、Data In 側の 4,5,7,8 ピンは、上流機器へ 24V が戻らないように PoE 電源へ接続しないこと。
- Passive PoE 出力にはヒューズ、PTC、または電流制限を入れ、ケーブル短絡時に過熱しないこと。定常電流は `B-DB-AC` の最大 7W を基準にし、起動突入とケーブル電圧降下を実測で確認する。
- Passive PoE では、PoE Out 側の RJ45 や基板シルクには、24V Passive PoE であり IEEE PoE ではないことを明示すること。
- E-Stop 押下、足回り・アーム出力遮断、足回り負荷急変時にも Wi-Fi アンテナモジュール出力が不要に落ち込まないこと。

### 5.4 常時給電と保護独立性

- 本基板は E-Stop 対象外とし、E-Stop 押下中も Mini PC と Wi-Fi アンテナモジュールへ常時給電できること。
- 入力バッテリーは足回り・アーム系と共有するが、出力スイッチ、保護、フィルタ、GND リターン、配線経路は常時給電系として分離すること。
- PDU の足回り・アーム ON/OFF、E-Stop、負荷変動に依存せず、Mini PC と Wi-Fi アンテナモジュールへ常時給電できること。
- PDU の常時給電枝が接続され、入力電圧、過電流、温度などの保護条件を満たす場合は、19V と Wi-Fi アンテナモジュール出力を自動で起動すること。
- 足回り・アーム高電力系の GND リターン電流が本基板の信号 GND、Ethernet、出力電源へ回り込みにくい配線にすること。
- RJ45 シールド、シャーシ、Ethernet 経由の絶縁条件は、ローバー全体のノイズと安全方針に合わせる。初期 PCB では 0ohm、RC、DNP を選べるフットプリントを用意する。

### 5.5 監視・表示

- MCU を必須としない単純な常時給電基板として成立すること。
- 入力電源有無、19V 出力有無、Wi-Fi アンテナモジュール出力有無、fault 状態を LED またはテストポイントで確認できること。
- 入力保護 IC や出力スイッチの power good、fault、current sense 信号は、将来 PDU MCU や Mini PC へ取り込めるようにテストポイントまたはヘッダへ出しておくこと。
- 初期試作では Mini PC へのシャットダウン通知やバッテリー残量通知は行わず、常時給電のみとする。
- 入力保護 IC や出力スイッチの telemetry/fault は、将来の低電圧予告やログ取得へ拡張できるようにテストポイントへ出しておく。

### 5.6 熱・EMC・配線

- 19V/5A 系は 100W 級として、MOSFET、インダクタ、電流検出抵抗、コネクタ、配線幅、銅箔厚、サーマルビアを見積もること。
- Wi-Fi アンテナモジュール出力は小電力でも、RJ45 や Ethernet データペアの近傍へスイッチングノードを寄せないこと。
- 19V DC/DC と Wi-Fi アンテナモジュール用 DC/DC のスイッチングノード、入力コンデンサ、GND リターンは短く太くし、Ethernet データペアから離すこと。
- 入力フィルタと TVS は、PDU 側の足回り・アーム高電力系が発生するサージ、リンギング、E-Stop 遮断過渡を考慮して選定すること。
- Ethernet データペアは 100ohm differential を意識し、1/2、3/6 のペア関係を崩さないこと。
- RJ45 シールドは直接 GND、RC 接続、シャーシ接続、未接続のどれにするかをローバー全体方針に合わせること。
- 屋外運用とローバー搭載を前提に、40 degC 以上の環境と筐体内温度上昇を見込むこと。

## 6. 非機能要件

- 電源投入時に過大な突入電流や不安定な再起動を起こさないこと。
- 入力低電圧時に DC/DC が発振的に再起動を繰り返さないこと。
- Mini PC と Wi-Fi アンテナモジュールの同時起動時でも、19V 出力が許容範囲を外れないこと。実機負荷 3.42A だけでなく、設計上限の 5A ダミー負荷でも評価すること。
- Passive PoE 出力は誤接続の危険が高いため、基板シルク、コネクタラベル、BOM メモで明確に識別すること。
- 高電流パスと Ethernet データペアはレイアウト上で明確に分離すること。
- 足回り・アーム系の負荷急変や故障時に、Mini PC と Wi-Fi アンテナモジュールが不要に再起動しないこと。
- 屋外の振動環境を考慮し、入力、Mini PC 出力、RJ45 は抜け止めまたは固定方法を検討すること。

## 7. 検証項目

- 確定した PDU 常時給電枝の入力下限、定格、上限で 19V/3.42A 実機負荷および 19V/5A ダミー負荷が安定する。
- 確定した PDU 常時給電枝の入力下限、定格、上限で `B-DB-AC` 用 24V 出力が 20V から 27V DC の許容範囲に入る。
- 19V/5A と Wi-Fi アンテナモジュール出力を同時負荷しても、入力保護が不要に遮断しない。
- E-Stop 押下中も 19V 出力と Wi-Fi アンテナモジュール出力が維持される。
- 足回り・アーム高電力系の ON/OFF、負荷急変、故障模擬時に 19V 出力と Wi-Fi アンテナモジュール出力が許容範囲を外れない。
- Mini PC 接続時の起動突入で 19V 出力が過大に落ち込まない。
- 外径 5.5mm / 内径 2.5mm、センタープラスの DC プラグで Mini PC へ正しい極性で給電できる。
- 19V 出力短絡時に入力ヒューズ、保護 IC、DC/DC、出力スイッチが破壊しない。
- Passive PoE で、PoE Out の 4,5 ピンに +24V、7,8 ピンに GND が出ている。
- Passive PoE で、Data In 側の 4,5,7,8 ピンへ 24V が戻っていない。
- Passive PoE で、Data In と PoE Out の 1,2,3,6 ピンが意図通り直結され、100Mbps link が成立する。
- Passive PoE で、10m から 20m 程度のケーブルで `B-DB-AC` を動作させ、起動突入、電圧降下、発熱が許容範囲に入る。
- Passive PoE で、1000BASE-T 機器を接続した場合に、設計上の非対応をラベルと仕様で確認できる。
- Passive PoE で、PoE ケーブル短絡時にヒューズまたは電流制限が動作し、RJ45、ケーブル、基板が過熱しない。
- 入力低電圧時に安全に停止し、復帰時に過大な突入を起こさない。
- 40 degC 以上の環境で、19V/5A 連続負荷時の MOSFET、インダクタ、出力スイッチ、コネクタ温度が許容範囲に入る。

## 8. 実装フェーズ

1. 要件確定
   - PDU 常時給電枝の入力電圧範囲、入力コネクタ、ヒューズ協調、`B-DB-AC` の起動突入、E-Stop 非連動条件を確認する。
2. 入力保護試作
   - PDU 常時給電枝からの受電、突入電流、低電圧遮断、過電圧保護、ヒューズ協調、ノイズフィルタを確認する。
3. 電源単体試作
   - 19V DC/DC と Wi-Fi アンテナモジュール用 DC/DC を個別に評価し、効率、発熱、起動、短絡保護を確認する。
4. E-Stop 非連動・ノイズ耐性評価
   - 足回り・アーム高電力系の ON/OFF、E-Stop、負荷急変、故障模擬時に常時給電が維持されることを確認する。
5. RJ45 / Passive PoE 統合
   - Passive PoE の pinout、ESD、100Mbps link、ケーブル短絡時の保護を確認する。
6. 実機負荷試験
   - Mini PC と `B-DB-AC` を同時接続し、起動、連続運用、温度、ノイズを確認する。
7. ローバー搭載評価
   - PDU 常時給電枝と接続したまま、E-Stop 非連動、Ethernet 接続、シールド、固定、耐振動を確認する。

## 9. 未定義として確定した要件

- PDU の常時給電枝から本基板へ渡す具体的な入力コネクタ型番、ピンアサイン、許容電流、ヒューズ協調、power good / fault 信号の電気仕様は未定義として確定する。PDU 側は BL4040 2 個系からの保護済み常時給電枝とし、専用ヒューズ、eFuse、フィルタ、TVS、power good / fault 線を持つ方針とする。
- `B-DB-AC` の実機起動突入電流、10m から 20m 程度の PoE ケーブルでの電圧降下、低温時の起動余裕は未定義として確定する。`B-DB-AC` の 20V から 27V DC、最大 7W、24V Passive PoE pinout は確定済みとし、突入電流とケーブル条件は試作で実測する。
- PDU の BL4040 低電圧警告、出力制限、遮断、復帰閾値が確定した後に設定する本基板の入力 UVLO、復帰閾値、Mini PC への低電圧予告要否は未定義として確定する。E-Stop 中の常時給電維持方針は確定済みとし、主バッテリー低電圧時の具体閾値は PDU 側の BL4040 実機評価に依存する。
