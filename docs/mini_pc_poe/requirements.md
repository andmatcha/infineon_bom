# Mini PC / Passive PoE 電源基板 要件整理

作成日: 2026-06-18  
入力: ユーザー要件および未確定要件への回答

## 1. 目的

Mini PC 用に、ローバー本体の PDU とは別系統のマキタ 18V バッテリーから 19V/5A を出力する DC/DC コンバーター基板を作る。接続する Mini PC 実機の定格は 19V/3.42A (64.98W) であり、基板側は起動突入と機種変更余裕を見込んで 19V/5A 定格として設計する。同じ基板に 24V/0.5A の Passive PoE インジェクタ機能も搭載し、Mini PC と周辺ネットワーク機器をローバー系電源から独立して常時給電できるようにする。

この文書では、PDU 要件整理と同様に、現時点で決まっている要件を本文へ反映して整理する。部品選定は [parts_selection.md](./parts_selection.md)、BOM は [../../bom/mini_pc_poe_bom.csv](../../bom/mini_pc_poe_bom.csv) に分ける。

## 2. 前提

- 本基板は、Mini PC および 24V Passive PoE 機器へ給電する独立電源基板である。
- 入力電源は、ローバー系 PDU とは別のマキタ 18V 系バッテリー 1 本を前提とする。
- ローバーの足回り系、アーム系、PDU 用バッテリーとは電源系統を共有しない。
- 出力は、基板定格 19V/5A の Mini PC 用電源と、24V/0.5A の Passive PoE 電源の 2 系統とする。
- Mini PC 実機の電源入力は 19V/3.42A、センタープラス、外径 5.5mm / 内径 2.5mm の DC ジャックである。
- 両出力は常時給電とし、バッテリー接続後、保護条件を満たす場合に自動で出力する。
- Passive PoE は IEEE 802.3af/at/bt 互換ではなく、検出・分類・ネゴシエーションを行わない固定 24V 給電とする。
- RJ45 の 1,2,3,6 ピンはデータ用として Data In と PoE Out の間を直結し、PoE Out 側の 4,5 ピンへ +24V、7,8 ピンへ GND を割り当てる。
- このピン割り当ては 10BASE-T / 100BASE-TX の spare-pair 給電を前提とする。1000BASE-T 以上は全 4 ペアをデータに使うため、この構成では対応しない。
- 電源保護、出力スイッチ、パワー MOSFET は Infineon を基本候補とする。DC/DC コントローラ、RJ45、コネクタ、ヒューズ、TVS、受動部品は Infineon 縛りの対象外とする。

## 3. スコープ

### 含めるもの

- マキタ 18V バッテリー入力
- 入力ヒューズ、逆接保護、突入電流制限、低電圧遮断
- 19V/5A Mini PC 用 buck-boost DC/DC
- 24V/0.5A Passive PoE 用 boost DC/DC
- Mini PC 出力および PoE 出力の過電流、短絡、過温度保護
- RJ45 Data In / PoE Out の配線、ESD 保護、シールド処理
- 電源状態確認用 LED、fault 信号、テストポイント
- 入出力コネクタ、ヒューズ、TVS、電源インジケータ

### 含めないもの

- Mini PC 本体の選定
- OS のシャットダウン制御
- IEEE 802.3af/at/bt 準拠の PoE PSE 機能
- 1000BASE-T 以上の Ethernet pass-through
- ローバー PDU、足回り、アーム電源との連動制御
- 最終 PCB レイアウト、熱シミュレーション、EMC 試験仕様

## 4. システム構成

```text
Makita 18V Battery C
  -> input fuse / reverse protection / hot-swap / UVLO
        -> protected input bus
        -> 19V buck-boost DC/DC
              -> high-side protection switch
              -> Mini PC 19V / 5A output
        -> 24V boost DC/DC
              -> high-side protection switch / fuse
              -> Passive PoE 24V / 0.5A output

Ethernet Data In RJ45
  pin 1 -> PoE Out pin 1
  pin 2 -> PoE Out pin 2
  pin 3 -> PoE Out pin 3
  pin 6 -> PoE Out pin 6

PoE Out RJ45
  pins 4,5 -> +24V
  pins 7,8 -> GND
```

ローバー系 PDU とは別バッテリー、別入力保護、別 DC/DC とし、足回り・アーム系の E-Stop や負荷変動に依存しない。Ethernet のデータ配線や RJ45 シールドを通じた接続ポリシーは、ローバー全体の GND/シールド方針に合わせて最終決定する。

## 5. 機能要件

### 5.1 バッテリー入力

- マキタ 18V 系バッテリー 1 本を入力とすること。
- バッテリー満充電から放電末期まで、19V/5A と 24V/0.5A の両系統を扱える入力範囲を持つこと。
- 入力電圧範囲の暫定設計値は 15V から 21V 程度とし、低電圧遮断閾値は 15.0V とする。
- 設計上限では 19V/5A が 95W、24V/0.5A が 12W であり、合計 107W 級となる。実接続 Mini PC では 19V/3.42A のため、PoE と合わせた通常負荷は約 77W 級となる。
- 変換効率と設計上限を含めて、入力側は 10A 以上の連続電流を扱えること。
- 入力に物理ヒューズを置くこと。
- 逆接、過電流、短絡、過温度、低電圧を検出または遮断できること。
- バッテリー挿抜時に下流の DC/DC 入力容量へ過大な突入電流を流さないこと。

### 5.2 Mini PC 19V 出力

- 出力電圧は 19V nominal とする。
- 出力電流は基板定格として連続 5A とする。接続する Mini PC 実機の定格負荷は 3.42A とする。
- 出力電圧許容差は Mini PC の要求仕様を優先する。初期目標は 19V ±5% とする。
- バッテリー電圧が 19V より高い場合、同等の場合、低い場合のいずれでも安定して 19V を生成できる buck-boost 構成とする。
- Mini PC 接続時の突入電流と出力容量充電を許容できること。
- 出力短絡、過電流、過温度、過電圧時に破壊しないこと。
- Mini PC 側コネクタは外径 5.5mm / 内径 2.5mm の DC ジャック、センタープラスとし、基板側からは同寸法のセンタープラス DC プラグ付きケーブルで給電すること。

### 5.3 Passive PoE 24V 出力

- 出力電圧は 24V nominal とする。
- 出力電流は連続 0.5A とする。
- 対象機器は 24V Passive PoE 対応が明記された機器に限定し、初期設計ではケーブル長 10m から 20m 程度を評価対象とする。
- 24V 出力は PoE Out RJ45 の 4,5 ピンへ +24V、7,8 ピンへ GND を供給すること。
- Data In RJ45 と PoE Out RJ45 の 1,2,3,6 ピンは、差動ペアとしてインピーダンスとペア関係を保って直結すること。
- Data In 側の 4,5,7,8 ピンは、上流機器へ 24V が戻らないように PoE 電源へ接続しないこと。
- Passive PoE 出力にはヒューズ、PTC、または電流制限を入れ、ケーブル短絡時に過熱しないこと。
- PoE Out 側の RJ45 や基板シルクには、24V Passive PoE であり IEEE PoE ではないことを明示すること。
- 対象機器が 24V Passive PoE 入力に対応していることを接続前に確認すること。

### 5.4 常時給電と独立性

- 本基板はローバー系 PDU とは完全に別のバッテリーから給電すること。
- ローバー系 PDU の ON/OFF、E-Stop、足回り/アーム負荷変動に依存せず、Mini PC と PoE 機器へ常時給電できること。
- バッテリーが接続され、入力電圧、過電流、温度などの保護条件を満たす場合は、19V と 24V を自動で起動すること。
- ローバー系 GND と本基板 GND を電源配線として共有しないこと。
- RJ45 シールド、シャーシ、Ethernet 経由の絶縁条件は、ローバー全体のノイズと安全方針に合わせる。初期 PCB では 0ohm、RC、DNP を選べるフットプリントを用意する。

### 5.5 監視・表示

- MCU を必須としない単純な常時給電基板として成立すること。
- 入力電源有無、19V 出力有無、24V PoE 出力有無、fault 状態を LED またはテストポイントで確認できること。
- 入力保護 IC や出力スイッチの fault / current sense 信号は、将来 MCU や Mini PC へ取り込めるようにテストポイントまたはヘッダへ出しておくことを推奨する。
- 初期試作では Mini PC へのシャットダウン通知やバッテリー残量通知は行わず、常時給電のみとする。
- 入力保護 IC や出力スイッチの telemetry/fault は、将来の低電圧予告やログ取得へ拡張できるようにテストポイントへ出しておく。

### 5.6 熱・EMC・配線

- 19V/5A 系は 100W 級として、MOSFET、インダクタ、電流検出抵抗、コネクタ、配線幅、銅箔厚、サーマルビアを見積もること。
- 24V/0.5A 系は小電力だが、RJ45 や Ethernet データペアの近傍へスイッチングノードを寄せないこと。
- 19V buck-boost と 24V boost のスイッチングノード、入力コンデンサ、GND リターンは短く太くし、Ethernet データペアから離すこと。
- Ethernet データペアは 100ohm differential を意識し、1/2、3/6 のペア関係を崩さないこと。
- RJ45 シールドは直接 GND、RC 接続、シャーシ接続、未接続のどれにするかをローバー全体方針に合わせること。
- 屋外運用とローバー搭載を前提に、40 degC 以上の環境と筐体内温度上昇を見込むこと。

## 6. 非機能要件

- 電源投入時に過大な突入電流や不安定な再起動を起こさないこと。
- 入力低電圧時に DC/DC が発振的に再起動を繰り返さないこと。
- Mini PC と PoE 機器の同時起動時でも、19V 出力が許容範囲を外れないこと。実機負荷 3.42A だけでなく、設計上限の 5A ダミー負荷でも評価すること。
- Passive PoE 出力は誤接続の危険が高いため、基板シルク、コネクタラベル、BOM メモで明確に識別すること。
- 高電流パスと Ethernet データペアはレイアウト上で明確に分離すること。
- 屋外の振動環境を考慮し、バッテリー入力、Mini PC 出力、RJ45 は抜け止めまたは固定方法を検討すること。

## 7. 検証項目

- 入力 15V、18V、21V で 19V/3.42A 実機負荷および 19V/5A ダミー負荷が安定する。
- 入力 15V、18V、21V で 24V/0.5A PoE 出力が安定する。
- 19V/5A と 24V/0.5A を同時負荷しても、入力保護が不要に遮断しない。
- Mini PC 接続時の起動突入で 19V 出力が過大に落ち込まない。
- 外径 5.5mm / 内径 2.5mm、センタープラスの DC プラグで Mini PC へ正しい極性で給電できる。
- 19V 出力短絡時に入力ヒューズ、保護 IC、DC/DC、出力スイッチが破壊しない。
- PoE Out の 4,5 ピンに +24V、7,8 ピンに GND が出ている。
- Data In 側の 4,5,7,8 ピンへ 24V が戻っていない。
- Data In と PoE Out の 1,2,3,6 ピンが意図通り直結され、100Mbps link が成立する。
- 10m から 20m 程度のケーブルで 24V Passive PoE 負荷を動作させ、電圧降下と発熱が許容範囲に入る。
- 1000BASE-T 機器を接続した場合に、設計上の非対応をラベルと仕様で確認できる。
- PoE ケーブル短絡時にヒューズまたは電流制限が動作し、RJ45、ケーブル、基板が過熱しない。
- 入力低電圧時に安全に停止し、復帰時に過大な突入を起こさない。
- 40 degC 以上の環境で、19V/5A 連続負荷時の MOSFET、インダクタ、出力スイッチ、コネクタ温度が許容範囲に入る。

## 8. 実装フェーズ

1. 要件確定
   - Mini PC 実機、DC プラグ寸法、PoE 対象機器、100Mbps 固定運用、低電圧遮断 15.0V を前提条件として確認する。
2. 電源単体試作
   - 19V buck-boost と 24V boost を個別に評価し、効率、発熱、起動、短絡保護を確認する。
3. 入力保護統合
   - バッテリー挿抜、突入電流、低電圧遮断、ヒューズ協調を確認する。
4. RJ45 / Passive PoE 統合
   - pinout、ESD、100Mbps link、ケーブル短絡時の保護を確認する。
5. 実機負荷試験
   - Mini PC と PoE 機器を同時接続し、起動、連続運用、温度、ノイズを確認する。
6. ローバー搭載評価
   - ローバー系 PDU とは独立したまま、Ethernet 接続、シールド、固定、耐振動を確認する。
