# Infineon CAN アームアクチュエータ要件整理

作成日: 2026-06-18  
入力: ユーザー要件および PDU 要件整理

## 1. 目的

University Rover Challenge (URC) に出場する ARES X のロボットアーム関節向けに、CAN 接続のスマートアクチュエータを開発する。STM32 MCU と Infineon 部品を中心に、BLDC モータ、サイクロイド減速機、エンコーダ、電流・電圧・温度監視を統合し、外部 CAN ノードから位置または速度を指定して駆動できる関節モジュールとして整理する。

この文書では、PDU 要件整理と同様に、現時点で決まっている要件を本文へ反映し、未定義として確定した要件を末尾に列挙する。部品選定は [parts_selection.md](./parts_selection.md)、BOM は [../../bom/arm_actuator_bom.csv](../../bom/arm_actuator_bom.csv) に分ける。

## 2. 前提

- 本アクチュエータは、University Rover Challenge (URC) に出場するために ARES Project が開発する新機体 ARES X のロボットアーム関節へ搭載することを前提とする。
- ARES X はローバー本体とロボットアームを含む火星探査ローバーであり、本アクチュエータはアーム関節の駆動、保持、状態監視を担う。
- 対象は、3 関節、手首ロール、ベースロールの 5 軸に加え、ベースの水平移動、把持を含む。
- 各駆動軸に 1 個のアクチュエータ制御基板を搭載し、同一設計を複数軸へ展開できることを目指す。
- 外部インターフェースは CAN を基本とし、上位の Arm MCU、Rover MCU、または Power Board と接続する。
- MCU は当プロジェクトで採用実績の多い STM32 を前提とし、モータ制御に必要な PWM、ADC、エンコーダ入力、CAN、SPI を持つ型番を選ぶ。
- モータは BLDC モータとし、サイクロイド減速機と併用する。
- モータドライバ、パワー MOSFET、CAN、角度センサ、電流センサ、電源監視は Infineon 部品を基本とする。STM32、コネクタ、ヒューズ、受動部品、機械部品、モータ本体は Infineon 縛りの対象外とする。

## 3. スコープ

### 含めるもの

- BLDC モータの三相インバータ駆動
- サイクロイド減速機を含む関節アクチュエータ制御
- ベース水平移動、把持など、関節回転以外のアーム駆動軸への展開
- CAN による指令受信、状態送信、故障通知
- 位置制御、速度制御、電流またはトルク相当の制限
- エンコーダによる角度取得
- モータ相電流、DC bus 電圧、基板温度、FET 近傍温度、モータ温度の監視
- 過電流、過電圧、低電圧、過温度、通信断、エンコーダ異常時の安全停止
- STM32 MCU、ゲートドライバ、CAN トランシーバ、監視電源

### 含めないもの

- BLDC モータ本体の詳細設計
- サイクロイド減速機の歯形、強度、潤滑、バックラッシュ設計
- アーム全体の逆運動学、軌道生成、干渉回避
- PDU 側の 24V 生成、アーム電源 ON/OFF、E-Stop 主遮断
- 最終 PCB レイアウト、熱シミュレーション、EMC 試験仕様

## 4. システム構成

```text
24V Arm Power from PDU
  -> input fuse / reverse protection / bulk capacitor
  -> DC bus voltage monitor
  -> 3-phase inverter
        -> BLDC motor
        -> cycloidal reducer
        -> arm joint output

STM32 MCU
  -> PWM / ADC / protection input
  -> Infineon 3-phase gate driver
  -> phase current sensing
  -> DC bus voltage sensing
  -> temperature sensing
  -> encoder / angle sensor
  -> CAN transceiver
```

初期試作では 24V 系の単一駆動軸アクチュエータとして構成し、電流レンジ、発熱、保持性能、減速機比を評価する。最終的には同一 CAN プロトコルと同一基板を複数駆動軸で使い、軸ごとの差分はモータ、減速比、電流制限、位置範囲、制御ゲイン、Node ID の設定で吸収する。

## 5. 機能要件

### 5.1 対象軸・基本性能

- ARES X のアーム駆動対象は、3 関節、手首ロール、ベースロール、ベースホライゾン、把持を含む。
- 最大可搬重量は 10kg とする。
- 第一関節は、過去機体の参考構成である RoboMaster M3508 クラスの BLDC モータと 1:120 サイクロイド減速機を基準に、出力定格トルク約 300N*m、最大出力トルク約 600N*m、出力回転数約 3.9rpm 相当を安定して発揮できることを目標とする。
- 第一関節以外の連続トルク、10 秒ピークトルク、100 ms ピークトルク、保持トルクは未定義として確定し、10kg 可搬時の姿勢ごとのモーメントから逆算して軸ごとに決定する。
- 肩、肘、手首、把持、ベースホライゾンでは必要トルクと速度が異なるため、同一制御基板を基本としつつ、モータ、減速比、MOSFET 並列数、電流制限、放熱は軸ごとに設定できること。
- 高速動作よりも、岩石の運搬、ボタン操作、把持、位置合わせを滑らかに行えることを優先する。
- 動作角度範囲は、URC のアームミッションを問題なく行えるだけの余裕を持ち、機械限界の内側に software limit を設定する。
- 各軸の最大速度、最大加速度、動作角度範囲、software limit の数値は未定義として確定し、岩石運搬、ボタン操作、把持、位置合わせの代表動作を決めた後に機械限界より内側へ設定する。

### 5.2 電源入力

- PDU のアーム 24V 系から給電されること。
- 入力電圧範囲は 24V nominal とし、PDU 側の 24V ±5% を受けられること。
- モータ駆動時の電流変動に耐える DC bus 容量を持つこと。
- 入力ヒューズ、逆接保護、突入電流対策、TVS などを基板仕様で検討する。
- DC bus 電圧を STM32 MCU が監視し、低電圧・過電圧時は安全側へ移行すること。
- 複数軸同時駆動時の総電流は PDU のアーム 24V 系要件へ反映し、単一アクチュエータの電流だけで PDU 側を決めないこと。
- PDU のアーム 24V 系へ要求する初期推定値は、連続 30A、10 秒ピーク 60A、100 ms ピーク 90A とする。各軸の実測値、電流制限、同時駆動パターンが決まり次第、この値を再見積もりする。

### 5.3 BLDC モータ駆動

- 三相 BLDC モータを駆動できること。
- BLDC モータは RoboMaster M3508 と同程度の 24V BLDC モータを基準候補とする。
- BLDC モータの具体型番、定格電流、ピーク電流、Kv、極対数、センサ有無は未定義として確定し、M3508 クラスを基準に、入手性、軸出力、熱余裕、制御しやすさを比較して最終型番を決める。
- 最終電流は軸ごとに未確定だが、第一関節では 1:120 サイクロイド減速機と組み合わせた高負荷運用を想定する。
- ゲートドライバは SPI などで設定と fault 読み出しができるものを優先する。
- 三相インバータの PWM は STM32 MCU が生成すること。
- 制御方式はセンサ付き FOC とする。
- ブレーキ、coast、disable、fault latch の各状態を明確に持つこと。

### 5.4 サイクロイド減速機との統合

- モータ出力はサイクロイド減速機を介して関節出力へ伝達する。
- 制御上はモータ軸角度、出力軸角度、減速比、バックラッシュ、弾性、摩擦を区別して扱う。
- 第一関節の基準構成は 1:120 サイクロイド減速機とし、出力定格トルク約 300N*m、最大出力トルク約 600N*m、出力回転数約 3.9rpm 相当を参考値とする。
- 第一関節以外の減速比、許容トルク、効率、バックラッシュ、自己保持性は未定義として確定し、必要速度とトルクから減速比を決め、バックラッシュと効率は試作で実測する。
- 出力軸側に角度センサを置くことを基本とし、必要に応じてモータ側ホールセンサまたはモータ側エンコーダを併用する。
- 電源 OFF 時と E-Stop 時の保持は減速機自己保持を前提とする。ただし、重力負荷が大きい軸ではブレーキ搭載も検討する。

### 5.5 エンコーダ・角度取得

- 関節の絶対角を取得できること。
- 初期候補は Infineon TLE5012B 系の磁気式 360 deg 角度センサとする。
- 角度センサは関節出力軸側に置くことを基本とする。
- エンコーダ情報はセンサ付き FOC、位置制御、速度推定、原点復帰、異常検出に使う。
- 起動直後に現在角度を取得し、通信開始前に角度値の妥当性を確認すること。
- 角度センサ異常、磁石位置ずれ、通信異常、角度ジャンプを検出した場合は FAULT 状態へ移行すること。

### 5.6 電流・電圧・温度監視

STM32 MCU は少なくとも以下を監視する。

- DC bus 電圧
- 三相シャントによる相電流
- 保護・ログ用の DC bus 電流
- ゲートドライバ fault 状態
- MOSFET 近傍 NTC 温度
- モータケース NTC 温度
- 角度センサの診断状態
- MCU 電源、リセット、ウォッチドッグ状態

電流制御に使う高速電流計測と、テレメトリ・保護に使う電流監視は分けて設計する。高速電流計測は PWM と同期した三相シャントの ADC 取得を基本とし、独立した DC bus 電流ログや過電流検出には Infineon TLE4971 または TLE4973 系の電流センサを併用する。

### 5.7 制御モード

各アクチュエータは以下の制御モードを持つ。

- `DISABLED`: ゲート駆動停止
- `CURRENT_LIMIT`: 電流またはトルク相当値を制限して駆動
- `VELOCITY`: 目標速度へ追従
- `POSITION`: 目標角度へ追従
- `HOLD`: 現在角度を保持
- `HOMING`: 原点または基準角を確認
- `FAULT`: 故障保持

位置制御では目標角度、最大速度、最大加速度、最大電流、タイムアウトを指令できること。速度制御では目標速度、最大電流、位置範囲制限、タイムアウトを指令できること。

### 5.8 CAN インターフェース

- 初期対応は CAN 2.0、ビットレートは 1Mbps を標準候補とする。
- CAN FD は将来拡張として扱う。
- 各アクチュエータは、関節位置または駆動軸に対応する固定 Node ID を持つこと。
- CAN では少なくとも以下のメッセージを扱う。
  - enable / disable
  - 位置指令
  - 速度指令
  - 電流制限またはトルク制限
  - 状態要求
  - テレメトリ送信
  - fault 通知
  - fault 解除要求
  - パラメータ読み書き
- テレメトリ送信周期は初期値 10 ms から 50 ms 程度とする。
- heartbeat timeout 初期値は 100 ms から 300 ms 程度とし、通信途絶直後は速度指令を 0 または `HOLD` にした後、一定時間復帰しない場合は関節の落下リスクを考慮して `DISABLED` またはブレーキへ移行する。

### 5.9 フェイルセーフ

- 電源投入時の初期状態は `DISABLED` とする。
- MCU リセット時に三相インバータが勝手に ON にならないこと。
- E-Stop または PDU 側のアーム電源遮断時に破壊的な逆起電力や過電圧を起こさないこと。
- 過電流、短絡、過温度、低電圧、過電圧、CAN 通信断、角度センサ異常、ゲートドライバ fault を検出したら安全側へ移行すること。
- 故障解除は、原因が解消され、上位ノードから fault clear 指令を受けた場合にのみ行うこと。
- ブレーキを搭載する軸と減速機自己保持だけでよい軸の分類は未定義として確定し、重力で落下すると危険な肩、肘などの高負荷軸はブレーキ搭載を優先検討し、手首や把持は負荷条件で判断する。
- 関節が重力で落下する可能性がある場合は、減速機自己保持を基本としつつ、電気的保持だけに依存しないブレーキまたは機械ストッパを検討すること。

### 5.10 パラメータ・ログ

- Node ID、軸名、軸種別、減速比、角度範囲、正転方向、電流制限、温度制限、制御ゲインを保存できること。
- パラメータは CAN 経由で読み出しでき、書き込みは安全状態でのみ許可すること。
- ログは、時刻、状態、指令値、実角度、実速度、電流、電圧、温度、fault 理由を含む。
- 初期試作では RAM ログまたは CAN テレメトリを優先し、不揮発ログは将来拡張として扱う。

## 6. 非機能要件

- 同一基板を複数駆動軸へ使えるよう、電流レンジとコネクタ仕様に余裕を持たせる。
- 高電流パスは配線抵抗、銅箔厚、コネクタ定格、温度上昇を見積もる。
- MOSFET、ゲートドライバ、シャント抵抗、電流センサは放熱経路を前提に配置する。
- エンコーダ配線、CAN、SPI、ADC はモータ相線と電源スイッチングノードから距離を取る。
- CAN はローバー内のノイズ環境を前提に、終端、GND、シールド、TVS、コモンモードチョークを検討する。
- モータと減速機の機械的バックラッシュや弾性を、制御ゲインだけで無理に隠さない。
- 室温試作だけでなく、40 degC 以上の環境と密閉時の温度上昇を見込む。

## 7. 検証項目

- 24V 入力で MCU、CAN、ゲートドライバ、角度センサが起動する。
- 電源投入直後に三相インバータが OFF のまま保持される。
- CAN 1Mbps で Node ID 指定の enable / disable ができる。
- センサ付き FOC で BLDC モータを安定して駆動できる。
- 位置指令に対して関節角度が指定範囲内で追従する。
- 速度指令に対して関節速度が指定範囲内で追従する。
- 電流制限を下げた場合、出力トルクまたは加速度が制限される。
- エンコーダ角度と機械的な出力角度が一致する。
- 第一関節相当の試験で、出力定格トルク約 300N*m、最大出力トルク約 600N*m、出力回転数約 3.9rpm 相当を安定して扱えるか評価する。
- 10kg 可搬相当の負荷条件で、岩石運搬、ボタン操作、位置合わせに必要な滑らかさを確認する。
- heartbeat timeout 時に安全側へ移行する。
- 過電流、短絡、低電圧、過電圧、過温度、角度センサ異常、ゲートドライバ fault を検出して `FAULT` へ移行する。
- PDU 側 E-Stop またはアーム電源 OFF 時に破壊的な電圧上昇がない。
- 複数ノードを同一 CAN bus に接続しても Node ID の衝突がない。

## 8. 実装フェーズ

1. 要件確定
   - 対象軸、必要トルク、速度、角度範囲、保持条件、モータ候補、減速比を確定する。
2. 単体インバータ試作
   - STM32、ゲートドライバ、MOSFET、電流計測、DC bus 監視を確認する。
3. BLDC 駆動試作
   - 無負荷モータで PWM、電流計測、fault、温度監視を確認する。
4. エンコーダ統合
   - 出力軸角度の取得、原点、方向、角度範囲を確認する。
5. 減速機統合
   - サイクロイド減速機と組み合わせ、速度、トルク、発熱、バックラッシュを確認する。
6. CAN 制御統合
   - 位置指令、速度指令、テレメトリ、fault 処理を実装する。
7. アーム実機統合
   - 複数関節、PDU、E-Stop、上位制御と組み合わせて検証する。

## 9. 未定義として確定した要件

- 第一関節以外の各駆動軸の連続トルク、10 秒ピークトルク、100 ms ピークトルク、保持トルクは未定義として確定する。第一関節は 300N*m 定格、600N*m ピーク相当を基準にし、他軸は 10kg 可搬時の姿勢ごとのモーメントから逆算する。
- 各駆動軸の最大速度、最大加速度、動作角度範囲、software limit の数値は未定義として確定する。岩石運搬、ボタン操作、把持、位置合わせの代表動作を決め、機械限界より内側に software limit を置く。
- RoboMaster M3508 と同程度とする BLDC モータの具体型番、定格電流、ピーク電流、Kv、極対数、センサ有無は未定義として確定する。M3508 クラスを基準に、入手性、軸出力、熱余裕、制御しやすさを比較して最終型番を決める。
- 第一関節以外の各軸のサイクロイド減速機の減速比、許容トルク、効率、バックラッシュ、自己保持性は未定義として確定する。第一関節は 1:120 を基準にし、他軸は必要速度とトルクから減速比を決め、バックラッシュと効率は試作で実測する。
- ブレーキを搭載する軸と、減速機自己保持だけでよい軸の分類は未定義として確定する。重力で落下すると危険な肩、肘などの高負荷軸はブレーキ搭載を優先検討し、手首や把持は負荷条件で判断する。
